`copyArrayAndMultiplyBy2`, `copyArrayAndDivideBy2`, `copyArrayAndAdd3`에 `myArray [1, 2, 3]`을 전달할 때, 원래 배열을 전달했을까요, 아니면 복사본을 전달했을까요? 원래 `myArray`에 대한 참조(Reference)를 전달했습니다. 그래서 함수 내부에서 새로운 빈 배열을 생성하고 변환된 값을 하나씩 복사한 것이 매우 중요했습니다. 입력 배열을 변경하지 않았습니다.

이는 의도적인 선택입니다. 이상적으로 함수는 <b>사이드 이펙트(Side Effect)</b>가 없어야 합니다. 사이드 이펙트란 함수 내부에서 일어나는 일이 반환값으로 정의되지 않은 외부에 영향을 미치는 것을 말합니다. 즉, 전달된 배열을 내부에서 변경하면 전역 데이터에 예상치 못한 영향을 미치게 됩니다. 반면, 배열을 전달받아 내부에서 새 배열을 만들고 반환하면, 함수의 유일한 결과는 출력 배열뿐입니다. 함수가 무엇을 할지 정확히 알 수 있습니다. 입력 배열을 받아 내부에서 새 배열을 만들고 반환합니다. 입력 배열은 변경되지 않으며, 이는 입력 배열이 전역으로 정의되어 있기 때문에 중요합니다. 함수의 동작은 반환값으로 완전히 정의되어야 하며, 실행할 때마다 전역 데이터에 예상치 못한 영향을 미쳐서는 안 됩니다. 반환값이 함수에서 나오는 유일한 결과물이어야 합니다.
