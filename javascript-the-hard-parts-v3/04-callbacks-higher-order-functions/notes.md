# JavaScript: The Hard Parts v3 - Callbacks & Higher-Order Functions

## 1. 콜백과 고차 함수 소개

<b>콜백(Callback)</b>과 함수의 조합은 자바스크립트에서 가장 오해가 많은 개념 중 하나입니다. 하지만 이를 제대로 이해하면 `map`, `filter`, `reduce` 같은 전문가 수준의 함수를 구현할 수 있습니다.

- `reduce`는 가장 유연한 함수로, 리듀스로 처리될 수 있으리라 생각하지 못했을 작업들도 실제로는 리듀스로 처리할 수 있습니다.
- 더 깊이 있는 내용은 <b>Functional Programming Hard Parts</b> 과정에서 다룹니다.
- 고차 함수(Higher-Order Functions)를 이해하면 코드를 더 <b>선언적(Declarative)</b>이고 <b>가독성</b> 높게 만들 수 있습니다.

---

## 2. 함수가 존재하는 이유

모든 것은 <b>"왜 함수가 필요한가?"</b>라는 질문에서 출발합니다.

이 과정에서는 예전 방식인 `function` 키워드를 사용하여 함수를 선언합니다. 화살표 함수(Arrow Function) 선언 방식도 있지만, `function` 키워드는 <b>가독성</b> 측면에서 함수가 있다는 것을 명확히 알 수 있어 교육용으로 특히 효과적입니다.

### 반복적인 함수 선언의 문제

10의 제곱을 구하는 함수를 작성해 봅시다:

```js
function tenSquared() {
  return 10 * 10;
}
tenSquared(); // 100
```

9의 제곱도 필요하다면?

```js
function nineSquared() {
  return 9 * 9;
}
```

8의 제곱도 필요하다면?

```js
function eightSquared() {
  return 8 * 8;
}
```

이렇게 매번 새로운 함수를 선언하는 것은 매우 비효율적입니다.

---

## 3. DRY 원칙

위와 같은 반복적인 함수 선언은 프로그래밍의 핵심 원칙인 <b>DRY(Don't Repeat Yourself)</b>, 즉 <b>"반복하지 마라"</b> 원칙을 위반하는 것입니다.

동일한 연산 로직을 값만 바꿔서 매번 새로 작성하는 것은 유지보수성과 확장성 모두에서 문제를 일으킵니다.

---

## 4. 매개변수를 통한 함수 일반화

DRY 원칙을 지키기 위한 해결책은 함수에 <b>매개변수(Parameter)</b>를 추가하여 <b>일반화(Generalize)</b>하는 것입니다.

```js
function square(num) {
  return num * num;
}

square(10); // 100
square(9);  // 81
square(8);  // 64
```

| 용어 | 설명 |
|------|------|
| <b>매개변수(Parameter)</b> | 함수 선언 시 정의하는 자리표시자(Placeholder). 나중에 실제 데이터로 채워집니다. |
| <b>인수(Argument)</b> | 함수를 실행할 때 제공하는 실제 입력값입니다. |

함수를 <b>한 번 저장</b>하고, 빈칸이 아닌 <b>레이블(Label)</b>을 남겨두면, 함수를 여러 번 실행할 때 <b>다른 데이터로 재사용</b>할 수 있습니다.

---

## 5. 기능까지 일반화하기: 고차 함수의 필요성

매개변수를 통해 <b>데이터</b>를 미정(TBD)으로 남겨둘 수 있다면, <b>코드(기능)의 일부</b>도 나중에 결정할 수 있도록 남겨둘 수 있지 않을까요?

- 함수를 한 번 저장합니다.
- 다른 데이터로 재사용합니다.
- 여기서 더 나아가, <b>기능(Functionality) 자체</b>도 나중에 결정할 수 있도록 남겨둘 수 있습니다.

이렇게 하면 더 <b>일반화 가능</b>하고 더 <b>재사용 가능</b>한 함수를 만들 수 있습니다.

| 일반화 수준 | 미정으로 남기는 것 | 예시 |
|-------------|---------------------|------|
| <b>기본 함수</b> | 없음 (모든 것이 고정) | `tenSquared()` |
| <b>매개변수화된 함수</b> | 데이터(Data) | `square(num)` |
| <b>고차 함수(Higher-Order Function)</b> | 데이터 + 기능(Functionality) | `map(arr, fn)` |

---

## 6. 고차 함수의 역사적 맥락

고차 함수의 개념을 당연하게 여기기 쉽지만, Java나 C++ 같은 주요 프로그래밍 언어에서는 <b>2010년대까지 이것이 불가능</b>했습니다.

- 함수가 정의된 후에 단순히 다른 데이터가 아닌 <b>다른 동작(Behavior)</b>을 부여하도록 조정하는 개념은 내장 기능이 아니었습니다.
- 어느 정도 우회적으로 구현할 수는 있었지만, 주요 언어들의 <b>내장 기능은 아니었고 나중에 추가</b>되었습니다.

<details>
<summary>Java와 C++에서 고차 함수가 추가된 시기</summary>

- **Java**: Java 8(2014년)에서 람다 표현식(Lambda Expressions)과 함수형 인터페이스(Functional Interfaces)가 도입되었습니다.
- **C++**: C++11(2011년)에서 람다 표현식이 도입되었습니다.

JavaScript는 처음부터 함수를 일급 객체(First-Class Object)로 취급하여, 함수를 다른 함수의 인수로 전달하거나 반환값으로 사용하는 것이 자연스럽게 가능했습니다.

</details>

따라서 <b>고차 함수(Higher-Order Functions)</b>를 가질 수 있다는 것을 당연하게 여기지 말아야 합니다. 고차 함수란 <b>함수를 호출할 때 비로소 기능의 일부를 결정</b>하는 것입니다.

## Q&A

<details>
<summary>일급 객체(First-Class Object)란 무엇인가요?</summary>

프로그래밍 언어에서 다른 값(숫자, 문자열 등)과 동일하게 취급되는 대상을 말합니다. JavaScript에서 함수는 일급 객체이므로 다음이 모두 가능합니다:

| 특성 | 예시 |
|------|------|
| 변수에 할당 | `const fn = function() {}` |
| 함수의 인수로 전달 | `arr.map(fn)` |
| 함수의 반환값으로 사용 | `return function() {}` |
| 자료구조에 저장 | `const arr = [fn1, fn2]` |

간단히 말해, <b>값처럼 자유롭게 다룰 수 있으면 일급 객체</b>입니다. JavaScript에서는 처음부터 함수가 일급 객체였기 때문에 고차 함수 패턴을 자연스럽게 사용할 수 있습니다.

</details>

<details>
<summary>이급 객체(Second-Class)나 삼급 객체(Third-Class)도 있나요?</summary>

공식 용어는 아니지만, 일급 객체와 대비하여 비공식적으로 구분하기도 합니다:

| 등급 | 의미 | 예시 |
|------|------|------|
| 일급(First-Class) | 변수 할당, 인수 전달, 반환값 사용 모두 가능 | JS의 함수, 숫자, 문자열 |
| 이급(Second-Class) | 인수로 전달은 가능하지만, 반환값이나 변수 할당에 제약 | 일부 언어의 함수 |
| 삼급(Third-Class) | 인수로 전달조차 불가능 | C의 레이블(label), 대부분 언어의 연산자(`+`, `-`) |

JavaScript에서는 함수가 완전한 일급 객체이기 때문에 이런 구분을 체감할 일이 거의 없습니다. 이 개념은 주로 언어 간 비교나 프로그래밍 언어론에서 등장합니다.

</details>

<details>
<summary>JavaScript에서 일급 객체가 아닌 것은 무엇인가요?</summary>

JavaScript에서는 사실상 거의 모든 것이 일급 객체입니다. 일급 객체가 아닌 것은 언어의 <b>구문 요소(Syntax)</b>들입니다:

| 일급 객체 (값으로 다룰 수 있음) | 일급 객체가 아닌 것 (값으로 다룰 수 없음) |
|------|------|
| 함수, 숫자, 문자열, 불리언, 객체, 배열, `null`, `undefined` | 연산자 (`+`, `-`, `===` 등) |
| | 제어문 (`if`, `for`, `while` 등) |
| | 선언 키워드 (`const`, `let`, `class` 등) |

예를 들어 `const myOperator = +;`나 `const myIf = if;`는 불가능합니다. 연산자나 제어문은 언어의 <b>문법(Syntax)</b>이지 <b>값(Value)</b>이 아니기 때문에, 변수에 할당하거나 함수에 전달할 수 없습니다.

</details>
