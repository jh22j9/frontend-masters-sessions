# Front-End System Design - Introduction

## 강사 및 과정 소개

본 워크숍은 **Evgenii Ray** 강사가 진행하며, 지난 10년간 업계에서 복잡한 UI 애플리케이션을 개발해 온 경력을 바탕으로 설계되었습니다. 특히 JetBrains에서의 3년 근무와 현재 Meta에서의 프론트엔드 엔지니어 경험을 토대로, UI 전문가들이 매일 사용하는 라이브러리와 프레임워크의 내부 동작 원리를 파악하는 데 집중합니다. 실무 적용 및 기술 면접에 필요한 근본적인 지식과 복잡한 패턴에 사용되는 API 설계를 학습하는 것이 목표입니다.

## 프론트엔드 시스템 디자인의 핵심 요소

애플리케이션 설계를 백엔드, API, 프론트엔드로 나눌 때, **Evgenii Ray** 강사가 정의하는 프론트엔드 시스템 디자인은 다음과 같은 영역을 포괄합니다.

- **API 통신 및 데이터 관리:** 서버와의 통신 방식 및 서버로부터 수신한 데이터를 관리하는 전략을 설계합니다.
- **디바이스 리소스 최적화:** 기기 메모리를 과도하게 점유하지 않도록 데이터를 효율적으로 저장하고 활용하는 방법을 다룹니다.
- **복잡한 사용자 상호작용 처리:** 제한된 RAM이나 작은 화면 환경에서 수천 개의 요소를 효율적으로 화면에 표시하는 등의 복잡한 인터랙션 문제를 해결합니다.
- **에셋 관리(Asset Management):** 복잡한 애플리케이션의 여러 청크(Chunks)를 효율적으로 로드하여 로딩 속도를 높이고, Web Vitals 성능 지표를 높은 수준으로 유지하는 전략을 포함합니다.

## 커리큘럼 세부 내용

1. **핵심 기초 (Core Fundamentals):** 브라우저가 HTML 요소를 처리하는 방식, Stacking Context(쌓임 맥락)와 Formatting Context(서식 맥락)의 연결 구조, 그리고 GPU와의 연동 과정을 다룹니다. 브라우저 렌더링 사이클과 합성 레이어(Composition Layers)를 살펴보고 GPU를 활용한 렌더링 최적화 방법을 학습합니다.
2. **DOM API:** 저수준(Low-level) API를 사용하여 DOM 트리를 수정하는 방법을 복습합니다. DOM API는 프론트엔드 개발의 가장 기본적인 도구로서 정확한 활용법 숙지가 필요합니다.
3. **Observer API:** 복잡한 패턴을 설계할 때 유용한 Observer API를 라이브 코딩 실습을 통해 학습하고, 실제 복잡한 디자인 패턴에 적용하는 방법을 확인합니다.
4. **가상화 (Virtualization):** 대규모 데이터를 효율적으로 처리하는 가상화의 동작 원리를 이해하고, 이를 바닥부터 직접 구현(Live-code)하는 심화 과정을 진행합니다.
5. **애플리케이션 상태 설계 (State Design):** 검색 및 접근 작업이 효율적으로 이루어질 수 있도록 상태를 설계하는 방법을 배웁니다. 기기 메모리 부하를 줄이기 위해 브라우저 저장소 API를 활용하는 전략을 다룹니다.
6. **네트워크 프로토콜:** 사용 가능한 다양한 통신 프로토콜과 UI 환경에서의 서버 통신 방식을 살펴봅니다. REST와 GraphQL의 차이점에 대한 분석과 **Evgenii Ray** 강사의 견해를 포함합니다.
7. **웹 애플리케이션 성능:** JavaScript, CSS, 이미지, 폰트 등 각종 에셋의 최적화 기법을 학습합니다.
8. **보너스 섹션 (인터뷰 시뮬레이션):** 실제 면접 환경을 가정하여 3D 형태의 애플리케이션 뉴스피드를 설계하는 실습을 진행합니다.

## 사전 준비 사항 (Prerequisites)

- **이론 지식:** 브라우저의 페이지 렌더링 방식 및 HTML 해석 과정에 대한 기본 이해, 기초적인 네트워크 지식.
- **프로그래밍 기술:** 기본적인 JavaScript 지식 (숙련도가 매우 높을 필요는 없음).
- **개발 환경:** 최신 버전의 크롬(Chrome) 브라우저. 본 과정은 별도의 번들러 없이 네이티브 환경에서 실습을 진행합니다.
