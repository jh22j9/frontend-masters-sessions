# Web Performance Fundamentals - User Expectations of Performance

## 1. 웹 성능이 중요한 이유 (Why Web Performance Matters)
Todd Gardner는 웹 사이트가 왜 빨라야 하는지에 대해 세 가지 핵심적인 이유를 제시합니다. 사이트의 성격에 따라 우선순위는 다를 수 있지만, 모든 웹사이트는 최소한 이 중 하나 이상의 영향을 받습니다.

1.  **사용자 경험 (User Experience)**: 사용자의 만족도와 직결되는 요소.
2.  **검색 엔진 최적화 (SEO)**: 검색 결과 상단 노출을 위한 필수 조건.
3.  **온라인 광고 (Online Advertising)**: 광고를 통해 유입된 사용자의 전환율 결정.

## 2. 시간대별 사용자 반응 임계점 (Response Time Limits)
사용자의 심리적 기대치는 학술적 연구(IEEE 등)에 기반하며, 지연 시간에 따라 다음과 같은 반응을 보입니다.

* **0.1초 미만 (Instant)**: 사용자는 지연을 전혀 느끼지 못하며 시스템이 즉각적으로 반응한다고 느낍니다.
* **1초 이내 (Seamless)**: 지연을 인지하지만 작업의 흐름(Flow)이 유지됩니다. 뇌가 다른 생각으로 전환되지 않는 수준입니다.
* **2초 (The Magic Number)**: 효과적인 커뮤니케이션을 위한 골든타임입니다. 2초를 넘기면 사용자는 즉시 '느리다'고 느끼며 집중력이 흐트러지기 시작합니다.
* **10초 이상 (Frustration)**: 거의 모든 사용자가 좌절감을 느끼고 집중력이 완전히 깨집니다. 인간과 컴퓨터의 상호작용이 중단되는 시점입니다.

## 3. 비즈니스 수익과의 상관관계 (Time is Money)
성능은 단순히 기술적인 수치가 아니라 실제 매출과 직결되는 비즈니스 지표입니다.

* **사용자 이탈 (Abandonment)**: 로딩이 **3초**를 넘어가면 사용자의 **40%**가 사이트를 떠납니다.
* **재방문 포기 (Lost Retention)**: 사이트가 느리다고 느낀 사용자의 **75%**는 다시는 그 사이트를 방문하지 않습니다.
* **신뢰도 하락**: 사용자는 느린 사이트를 신뢰하지 않으며, 이는 이커머스 등 수익형 모델에서 치명적인 타격을 줍니다.

## 4. 조교의 요약
* 웹 성능은 사용자가 기대하는 **속도와 효율성**을 충족시키는 과정입니다.
* 우리가 모든 유저에게 빠른 인터넷이나 최신 폰을 사줄 수는 없으므로, **웹사이트 자체의 최적화**가 유일한 해결책입니다.
* 사용자의 집중력을 붙잡아두기 위해서는 최소 **2초 이내**의 응답을 목표로 해야 합니다.
