# Web Performance Fundamentals - Introduction

## 1. 강사 소개: Todd Gardner
Todd Gardner는 단순한 개발자를 넘어 웹의 신뢰성과 속도를 수치화하고 개선하는 '웹 성능 전문가'입니다.

* **경력 및 활동**
    * 2002년부터 소프트웨어 엔지니어로 활동한 20년 이상의 베테랑.
    * 2017년부터 Frontend Masters 강사로 활동하며 다수의 성능 최적화 강의 진행.
    * 현재 컨설턴트 및 전문 트레이너로 대기업과 컨퍼런스에서 성능 최적화 교육 담당.

* **창업 및 기술 전문성**
    * **TrackJS 창업**: 자바스크립트 에러 모니터링 서비스로, 수만 개의 앱에서 발생하는 자바스크립트 오류 패턴과 해결책 분석.
    * **Request Metrics 창업**: 실시간 웹 성능 모니터링(RUM) 도구로, 전 세계 웹사이트가 느려지는 원인을 데이터 기반으로 진단.
    * 지난 6년간 수천 개의 실제 운영 환경 데이터를 분석하며 얻은 '실무에서 즉시 사용 가능한 성능 개선 노하우'를 본 과정에 압축함.

## 2. 워크숍 커리큘럼
본 워크숍은 이론에 그치지 않고 도구 활용과 데이터 해석에 집중합니다.

* **Why Performance?**: 성능이 비즈니스 수익과 사용자 경험에 미치는 영향.
* **Measuring (핵심)**: 다양한 메트릭(LCP, FID, CLS 등)의 의미와 데이터 수집 방법.
* **Testing & Tools**: Chrome DevTools, Lighthouse 등 성능 측정 도구 실습.
* **Goal Setting**: 서비스 성격에 맞는 현실적인 속도 목표 설정.
* **Improving**: 각 지표를 개선하기 위한 구체적인 프론트엔드 최적화 전술(Tactics).
* **Bonus Contents**:
    * 워터폴(Waterfall) 및 플레임(Flame) 차트 정밀 분석법.
    * 성능 데이터 해석을 위한 기초 통계 및 데이터 리터러시.

## 3. 실습 환경 준비
강의의 실무 예제를 직접 실행하기 위해 다음 환경이 필요합니다.

    * **설치 방법**:
        1. 레포지토리 클론 (git clone)
        2. 의존성 설치 (npm install)
        3. 로컬 서버 실행

## 4. 학습 목표
이 과정의 목표는 단순히 '코드를 빠르게 짜는 법'을 배우는 것이 아닙니다. 
1. 무엇이 성능을 저해하는지 **정확히 측정**하고, 
2. 복잡한 성능 차트에서 **병목 현상을 발견**하며, 
3. 데이터를 근거로 **최적의 개선안을 도출**하는 능력을 갖추는 것입니다.
