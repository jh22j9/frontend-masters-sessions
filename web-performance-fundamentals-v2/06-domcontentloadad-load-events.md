# Web Performance Fundamentals - Measuring DOMContentLoaded & Load Events

## 1. 레거시 지표란?
"레거시"라는 표현은 이 지표들이 사라진다는 의미가 아니라, 과거에 웹 성능을 측정할 때 주로 사용하던 지표라는 의미입니다. 현재는 예전만큼 많이 쓰이지 않지만, 여전히 이해하고 있어야 합니다.

## 2. DOMContentLoaded (DCL)
HTML이 완전히 다운로드되고 파싱이 완료되어, HTML 안에 포함된 모든 **스크립트(JavaScript)가 완전히 실행**된 상태입니다.

* DOM 요소에 접근 및 조작이 가능한 시점을 파악할 수 있습니다.
* 이미지는 아직 로드되지 않았을 수 있습니다.

워터폴 차트에서 파란 점선(DCL 마커)은 JavaScript 실행이 완료된 직후, 이미지 로드가 끝나기 전에 위치합니다.

```
[HTML 다운로드] → [CSS 다운로드] → [JS 다운로드 & 실행] → | DCL |
                                                              ↑
                                              이 시점에 이미지는 아직 로드 중일 수 있음
```

아래는 geocities.com의 필름스트립 예시입니다.

| 시점 | 상태 |
|------|------|
| 탐색 시작 | 빈 화면 |
| 파싱 진행 중 | 콘텐츠 일부 렌더링 |
| **DCL 발생** | **구조 완성, 이미지 미완성** |
| DCL 이후 | 이미지 로드 완료 |

페이지의 모든 DOM 요소에 접근 및 조작이 가능한 시점을 파악하기 위해 사용합니다.

```js
window.addEventListener('DOMContentLoaded', (event) => {
  console.log(event.timeStamp); // 예: 1807ms
});
```

## 3. Load Event
HTML과 모든 **알려진(known) 리소스**가 다운로드되고 렌더링이 완료된 시점입니다. 즉, 이미지를 포함한 모든 리소스가 페이지에 표시된 상태입니다.

"알려진(known)" 리소스의 구분은 다음과 같습니다.

| 경우 | Load 이벤트와의 관계 |
|------|----------------------|
| JavaScript로 동적 추가 (Load 이전) | Load 이벤트가 기다림 |
| JavaScript로 동적 추가 (Load 이후) | 이미 발생한 이후이므로 무관 |
| `loading="lazy"` 로 지정된 리소스 | Load 이벤트 이후에도 로드될 수 있음 |

워터폴 차트에서 빨간 마커(Load)는 가장 끝, 마지막 이미지가 완료된 지점에 위치합니다.

```
[HTML] → [CSS] → [JS] → [이미지1] → [이미지2] → ... → | Load |
                                                          ↑
                                              마지막 이미지 완료 시점
```

* 이 시점에 브라우저의 **로딩 스피너가 멈춥니다.**
* 단, 실제로 페이지가 완전히 완료됐다는 의미는 아닙니다. JS 동적 작업 등이 계속될 수 있습니다.
* 초기 페이지 설정 이후의 **리포팅, 업데이트 작업 시작** 기점으로 활용합니다.

```js
window.addEventListener('load', (event) => {
  console.log(event.timeStamp); // Load 이벤트 발생 시점 (ms)
});
```

## 4. 두 지표 비교

| 지표 | 완료 조건 | 이미지 포함 | 주요 용도 |
|------|-----------|-------------|-----------|
| **DOMContentLoaded** | HTML 파싱 + JS 실행 완료 | ❌ | DOM 접근 가능 시점 파악 |
| **Load** | 모든 알려진 리소스 로드 완료 | ✅ | 페이지 초기화 완료 시점 파악 |
